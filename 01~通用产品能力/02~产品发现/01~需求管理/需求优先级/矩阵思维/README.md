# 矩阵思维

![KANO 思维模型](https://image.woshipm.com/wp-files/2018/02/dFE2KsiigC7SdvO405Ea.png)

## 一、核心背景

1. 需求管理是产品经理核心工作，完整流程为：需求收集 → 需求分析 → 确定优先级 → 需求到功能 → 功能设计 → 需求评审 → 开发上线 → 效果评估。
2. 核心指导观点：俞军 PM12 条提到“决定不做什么比决定做什么重要”，凸显需求优先级确定的关键意义。

## 二、常见需求优先级确定方法

1. **KANO 模型**：以“需求实现度”和“用户满意度”为维度，将需求分为兴奋型、期望型、无差异型、基本型、反向型 5 类，按类型制定应对策略。
2. **四象限法则**：以“重要”和“紧急”为维度，将需求分为重要且紧急、重要不紧急、不重要但紧急、不重要也不紧急 4 类，对应不同处理优先级。
3. **时间评估与性价比**：核心公式为“性价比=商业价值／开发量”，通过打分比较性价比，同时需结合其他因素综合判断。

## 三、方法本质与实际应用

1. **本质**：上述方法核心是矩阵思维，即通过象限归类将无序需求转化为有序，辅助决策。
2. **应用案例**
   - 开发 MVP：聚焦“时间成本、是否为核心功能、能否展现产品特色”三维度，按组合判断需求优先级。
   - 处理老板需求：围绕“提起次数、是否有必要做、时间成本”三维度，制定差异化处理方案。

## 四、总结

1. 现有模型可解决多数优先级问题，但存在特殊场景局限性。
2. 理解矩阵思维本质，能帮助产品经理灵活适配项目、团队、公司现状，制定最优需求优先级方案。
