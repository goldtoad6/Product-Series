# 杨堃《决胜 B 端：驱动数字化转型的产品经理（第 2 版）》章节关键内容总结

全书共分**五篇（概述篇、设计篇、管理篇、进阶篇、成长篇）+ 附录**，核心围绕“B 端产品从 0 到 1 建设及数字化转型”展开，贯穿“M 公司分销平台”实战案例，第二版新增约 210 页内容（重点补充产品商业化论述），重写多章节，整体更侧重实战性与商业化视角。

## 一、概述篇（第 01-02 章）：建立 B 端产品认知框架

### 第 01 章 互联网产品领域探秘（较大调整）

1. **产品经理核心定义**

   - 核心原则：必须“定义问题、解决问题、为价值交付负责”，区别于传统 BA（仅负责需求传递）、项目经理（仅负责项目交付）。
   - 泛化定义：国内数字化浪潮下，“通过数字技术赋能业务的人”均被统称产品经理（含自研内部系统设计者）。

2. **产品分类与 B 端产品定位**

   - 互联网产品分 5 类：C 端、B 端、数据与策略、商业变现、AI 产品；乙方软件公司多统称 B 端产品（含 SaaS、数据产品）。
   - B 端产品本质：服务企业/组织，解决经营管理问题，核心目标是**提收入、升效率、降成本、控风险、保品质**。
   - B 端产品核心特点：用户是“群体”（需协作）、用户 ≠ 客户（采购决策人与使用者分离）、效能优先于体验、强调抽象与逻辑、收益难量化。

3. **价值链模型与 B 端产品关联**
   - 结合迈克·波特价值链模型：业务型产品覆盖“基本活动”（如 CRM 支撑市场销售、WMS 支撑出货物流）与“辅助活动”（如 HRM 支撑人力资源）；工具型产品为细分场景补充效率（如电子签名、企业网盘）。

### 第 02 章 B 端产品建设概述（内容新增+案例引入）

1. **B 端产品建设核心流程**

   - 三阶段：**市场分析与业务调研 → 产品方案设计（整体+细节）→ 落地优化（技术设计+实施+运营）**，需“自顶向下”设计（如盖大楼，需详尽规划）。
   - 关键差异：自研自用产品侧重“业务适配”，商业化产品需额外关注“市场定位与盈利模式”；对比 C 端（如盖平房，可快速调整）。

2. **B 端产品分类（按特点）**

   - 业务型：复杂度最高，支撑多人协作达成业务目标（分垂直业务型如 ERP/SCM，办公协作型如 OA/HRM）；
   - 工具型：聚焦单一明确场景（如视频会议、电子签名），偏个体使用，业务链条短；
   - 交易型：支撑企业交易线上化（如电商系统、OMS 订单系统），含 C 端获客与后台管理闭环；
   - 基础服务型：抽象公共能力（如 API 组件、数据中台），支撑上层产品。

3. **实战案例引入**
   - 引出贯穿全书的“M 公司分销平台”案例，明确其定位：从自研自用逐步转向商业化，目标是“农产品流通领域数字化解决方案”。

## 二、设计篇（第 03-08 章）：B 端产品设计全流程（重写/新增核心章节）

### 第 03 章 B 端产品的市场分析与业务调研（重写绝大部分）

1. **市场分析：外部+内部双视角**

   - 外部视角（商业化产品核心）：
     - 产业链分析：行业一体化解决方案需深入（如明源云研究房地产产业链），工具型产品无需；重点关注“不同环节的痛点、利润率、IT 预算”（如农产品产业链聚焦“产地批发商-物流-销地批发商”）；
     - 市场规模：直接数据缺失时，可通过“客户业务规模 ×IT 投入占比”估算（如农产品渠道商万亿规模 ×0.01% IT 投入=1 亿分销软件市场）；
     - 竞争格局：用行业集中度（CRn）判断市场竞争程度（如枣核形市场适合 SaaS 产品，腰部企业付费能力强、需求收敛）。
   - 内部视角（所有 B 端产品通用）：调研典型客户/对口业务方，理解“业务原理、流程、规则”，为功能设计提供依据。

2. **业务调研方法与案例**
   - 流程：明确目标 → 选调研对象 → 定方法（访谈、轮岗、问卷）→ 执行 → 总结；
   - 案例：M 公司分析农产品产业链，通过行业报告（如 ×× 网）梳理流通图，锁定“批发商”为核心客群；
   - 关键能力：产品经理需具备高效信息检索能力（获取行业报告、数据）。

### 第 04 章 B 端产品的整体方案设计（深化实操路径）

1. **整体方案核心模块**

   - 核心业务流程：梳理主干流程，确定“需线上化的环节”（如 M 公司分销平台的“采购-物流-批发”环节）；
   - 产品定位：宏观（目标客群+核心价值），执行层（子系统划分+用户群体+业务目标）；
   - 应用架构：规划与外部系统的融合关系（如对接 ERP、支付系统）；
   - 功能模块：抽象核心功能（如分销平台的“供应商管理、订单管理、库存管理”）；
   - 演进蓝图：按业务优先级规划实施节奏（先核心功能，后拓展功能）。

2. **协作要求**：需业务负责人、技术负责人参与，确保对方案的共识。

### 第 05 章 B 端产品的细节方案设计（重写+完善数据建模）

1. **核心：业务建模（数据+流程）**

   - 数据建模：抽象业务实体与关系（如客户、订单、商品），保证系统灵活性（避免后期重构）；
   - 流程建模：梳理业务流转逻辑（如订单审核流程），适配多人协作场景。

2. **关键细节设计**
   - 界面设计：新增“常见组件”讲解，遵循尼尔森十大可用性原则（效能优先，兼顾体验）；
   - 权限设计：功能权限（按钮/菜单）、数据权限（数据可见范围）、RBAC 模型（角色绑定权限）；
   - 报表设计：支撑业务监控（如销售数据报表），需贴合业务决策需求；
   - 文档：PRD 编写规范，确保技术与业务理解一致。

### 第 06 章 基于场景的需求分析和建模（绝大部分新增）

- 核心：现代软件设计“从场景挖掘到建模”的全流程，提出“需求挖掘十三要素五步法”，强调“洞察需求本质”（如用户说“要报表”，本质是“需监控库存周转”），避免表面需求。

### 第 07 章 以用户为中心的体验设计（大部分新增）

- 突破“B 端无需体验”误区：覆盖人机交互设计、服务蓝图（梳理用户与系统的触点）、用户体验分层（功能层、体验层、情感层），确保“高效能用”且“体验不糟”。

### 第 08 章 B 端产品的产品化（全部新增）

- 核心：平衡“标准化”与“个性化”，提出产品化三层次（基础功能标准化、个性化需求配置化、特殊需求定制化）；
- 关键概念：解析 aPaaS、低代码平台的价值（降低定制成本，提升规模化能力）；
- 目标：解决 B 端产品“项目制陷阱”，实现可复用、可规模化。

## 三、管理篇（第 09-12 章）：产品落地与持续迭代

### 第 09 章 B 端产品经理与技术方案（基本无改动）

- 技术认知边界：产品经理无需写代码，但需理解“MVC 范式、接口调用、数据库 SQL、软件工程‘搭积木’逻辑”，以便与技术团队高效协作（如判断需求技术可行性）。

### 第 10 章 B 端产品的项目管理与研发交付（重写绝大部分）

- 核心：经典软件工程与项目管理框架（如瀑布式、敏捷），覆盖“成本控制、进度管理、质量保障”，解决“复杂项目推进难、跨团队协作低效”问题（如 M 公司分销平台的研发排期）。

### 第 11 章 B 端产品的运营管理（一半新增）

- 分场景运营：
  - SaaS 产品：关注用户续费、活跃度（如培训、客户成功体系）；
  - 自研系统：关注用户 Adoption（如推广使用、问题反馈收集）；
- 核心目标：确保产品上线后持续产生业务价值，而非“上线即终点”。

### 第 12 章 B 端产品的需求管理与迭代优化（绝大部分新增）

- 需求管理细节：需求定义（区分业务需求、用户需求、功能需求）、优先级排序（如四象限法）、效果评估（数据驱动，如“提效 30%”需量化验证）；
- 迭代逻辑：通过“需求池-评审-开发-上线-复盘”循环，持续优化产品（如根据 M 公司用户反馈调整分销平台的库存预警功能）。

## 四、进阶篇（第 13-16 章）：企业级应用架构

- 核心：从“单一产品”升级到“体系化产品”，分析企业应用架构随业务的演进规律（如从小门店到集团企业的系统架构变化）；
- 关键观点：不单独设“中台”章节，认为中台本质属于“应用架构范畴”，需结合业务规模按需建设（避免“为中台而中台”）；
- 目标：帮助产品经理从“业务视角”升级到“企业经营视角”，设计支撑长期发展的产品体系。

## 五、成长篇（第 17 章）：B 端产品经理职业发展（大部分新增）

1. **能力模型**：业务理解能力（核心）、商业分析能力、技术认知能力、项目管理能力、沟通协作能力；
2. **成长路径**：自学建议（如读行业报告、参与业务轮岗）、学习资源推荐（书籍、课程）；
3. **职业壁垒**：强调“业务+技术+商业”的复合型经验，区别于 C 端产品经理的“流量运营”能力。
